Documentazione API Whisper AI Service

Benvenuto nella documentazione dell'API del Whisper AI Transcription Service. Questo servizio offre un modo semplice e potente per trascrivere file audio in testo utilizzando i modelli di OpenAI Whisper.

Versione: 1.0.0
Host: https://transcription.wraiter.it
Modello di base: Whisper AI (base)

Panoramica

Il servizio Whisper AI Transcription è un microservizio basato su FastAPI che utilizza OpenAI Whisper per trascrivere file audio in testo. Il servizio accetta file audio in vari formati e può riconoscere automaticamente la lingua del contenuto, con supporto ottimizzato per l'italiano.

Questa API è progettata per essere intuitiva e facile da integrare in qualsiasi applicazione che necessiti di funzionalità di trascrizione audio.

Autenticazione

Attualmente, il servizio non richiede autenticazione ed è accessibile pubblicamente. Tuttavia, è soggetto a rate limiting per prevenire abusi.

Rate Limiting

Per garantire la disponibilità del servizio per tutti gli utenti, sono in vigore i seguenti limiti di utilizzo:

  • Massimo 10 richieste al minuto per IP
  • Dimensione massima file: 25MB
  • Durata massima audio: 10 minuti

Codici di Errore

L'API utilizza codici di stato HTTP standard per indicare il successo o il fallimento di una richiesta. Ecco i principali codici di errore che potresti incontrare:

Codice Descrizione
200 Successo. La richiesta è stata elaborata correttamente.
400 Richiesta non valida. Potrebbe essere dovuto a parametri mancanti o file non validi.
500 Errore interno del server. Si è verificato un problema durante l'elaborazione della richiesta.

API Reference

Questa sezione descrive in dettaglio gli endpoint disponibili nell'API e come utilizzarli.

POST /transcribe/

Questo endpoint accetta un file audio e lo trascrive in testo. Supporta vari formati audio come MP3, WAV, M4A, FLAC e altri.

Parametri della Richiesta
Nome Tipo Richiesto Descrizione
audio_file File Sì Il file audio da trascrivere. Deve essere un formato audio valido.
language String No Codice della lingua dell'audio (es. "it" per italiano). Se non specificato, Whisper tenterà di rilevare automaticamente la lingua.
Risposte
200 OK

La trascrizione è stata completata con successo.

{
  "transcription": "Testo trascritto dal file audio."
}
400 Bad Request

Parametri mancanti o non validi.

{
  "detail": "Il file caricato non è un file audio"
}
500 Internal Server Error

Si è verificato un errore durante l'elaborazione della richiesta.

{
  "detail": "Errore durante la trascrizione: [messaggio di errore]"
}

Formato della Risposta

Tutte le risposte sono in formato JSON e includono i seguenti campi:

Campo Tipo Descrizione
transcription String Il testo trascritto dal file audio. 
{
  "transcription": "Questo è un esempio di testo trascritto da un file audio in italiano."
}

Esempi di Richieste

Esempio con cURL 
curl -X POST "https://transcription.wraiter.it/transcribe/" \
  -H "accept: application/json" \
  -H "Content-Type: multipart/form-data" \
  -F "audio_file=@/percorso/al/file_audio.mp3" \
  -F "language=it"
Esempio con JavaScript (Fetch API) 
const formData = new FormData();
formData.append('audio_file', fileInput.files[0]);
formData.append('language', 'it');

fetch('https://transcription.wraiter.it/transcribe/', {
  method: 'POST',
  body: formData
})
.then(response => response.json())
.then(data => {
  console.log('Trascrizione:', data.transcription);
})
.catch(error => {
  console.error('Errore:', error);
});
Esempio con Python (requests) 
import requests

url = "https://transcription.wraiter.it/transcribe/"

files = {
    'audio_file': open('file_audio.mp3', 'rb')
}

data = {
    'language': 'it'
}

response = requests.post(url, files=files, data=data)
result = response.json()

print("Trascrizione:", result['transcription'])
